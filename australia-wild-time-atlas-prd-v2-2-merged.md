## 0. 本次融合说明（必须先读）

本 PRD 已将新提出的“卡通绘本扩充版”要求，融合进现有的“地图优先、时间轴驱动、全国默认态”产品框架中。  
融合后的结果，不是简单叠加，而是经过取舍后的统一方案。

### 0.1 新要求中被正式吸收的内容
以下内容已正式纳入本 PRD：

1. **视觉风格整体改向**
   - 从“数字自然博物馆 / 深色展陈”路线，调整为：
   - **治愈系、手绘绘本、羊皮纸、低饱和大地色、全年龄友好的互动自然绘本网页**

2. **地图底图风格改为手绘绘本地图**
   - 地图应像一张铺开的羊皮纸 / 水彩地图
   - 不能再走深海蓝、赛博、仪表盘式视觉路线

3. **动物表现层改为 2D 卡通 / 手绘插画**
   - 禁止用原生 emoji 作为主动物图层
   - 改用统一风格的小型动物插画 / 剪影 / 绘本 icon

4. **时间轴事件表达更具叙事感**
   - 灭绝类物种可采用：
     - 线稿化
     - 褪色
     - 幽灵态
     - 擦除消散
   - 扩张类物种可采用：
     - 小碎步移动
     - 柔和路径位移
     - 节点增加

5. **首屏序章改为“翻开绘本”**
   - 从之前的“展陈导语”转为“翻开一本自然绘本”的微动画
   - 序章时长从 30 秒收缩为：**不超过 10 秒**
   - 更符合大众网页停留习惯，也更符合 MVP

6. **物种阵容扩容到 18 种**
   - 已纳入本 PRD 的 MVP 样本池
   - 但会做分层加载和同屏密度控制，避免性能失控

### 0.2 新要求中被修正后吸收的内容
以下要求方向正确，但在执行上需要收敛，因此已做修正后纳入：

1. **“近两千年演化”**
   - 产品表达上可以保留“长时段”叙事感
   - 但 MVP 的严格时间轴执行范围仍以：
     - `1788前自然状态（推断）`
     - `1788`
     - `1900`
     - `1950`
     - `Now`
     为主骨架
   - 用户层支持逐年滑动
   - 底层仍采用关键帧插值
   - 不做“公元前到现代的完整逐年事实数据库”

2. **“儿童/全年龄段绘本感”**
   - 采纳“治愈、温暖、亲和、可故事化”
   - 但不做过幼态化的幼儿园风格
   - 目标仍是：
     - 成年大众也愿意停留
     - 科普内容仍然可信
     - 页面不显得低龄廉价

3. **“物理回弹感”**
   - 采纳，但必须克制
   - 只用于：
     - 面板出现
     - 点击动物
     - 地图聚焦放大
   - 不允许全站所有元素都果冻抖动
   - 不允许影响稳定感和阅读性

### 0.3 新要求中不直接照搬的部分
以下内容不会按字面完全执行，而是按 MVP 现实进行限制：

1. **“全部动物都以复杂卡通动画持续运动”**
   - 不执行
   - 原因：18 个物种 × 多实例 × 时间轴逐年滑动，会明显增加掉帧风险
   - 替代方案：
     - 默认态使用“轻待机动效 + 局部迁徙动效”
     - 只给迁徙型、事件型物种更强的动画

2. **“所有动物都在每个年份都有丰富行为变化”**
   - 不执行
   - 原因：MVP 不应走向高成本动画片工程
   - 替代方案：
     - 关键年份变化显著
     - 中间年份平滑过渡
     - 重点物种重点演出

## 1. 项目概述

### 1.1 产品一句话定义
一个以澳大利亚为范围、以时间轴为主控制器、以全屏手绘风地图为页面本体的互动式数字自然绘本网页。

### 1.2 产品目标
让用户进入网页后，不需要先学习复杂操作，也不需要跳转多个页面，而是直接在一张充满生命力的澳大利亚绘本地图上，通过拖动时间轴，直观感受到动物的迁徙、扩张、收缩、灭绝与当下状态。

### 1.3 产品本质
本产品不是：
- GIS 数据平台
- 动物百科搜索站
- 教育后台
- 复杂地图分析工具

本产品是：
- **地图驱动的科普叙事网页**
- **时间驱动的生命演化体验**
- **绘本式互动自然展陈**

### 1.4 核心体验流程
1. 进入网站，触发不超过 10 秒的“翻开绘本”序章动画
2. 进入全国默认态地图
3. 直接看到多种动物在澳大利亚地图上的可视化存在
4. 拖动底部时间轴，观察动物的变化
5. 点击地图中的州区域，镜头推进放大
6. 点击具体动物，查看故事与详情
7. 时间轴停在 Now 时，阅读完整动物卡片并播放音频

## 2. 产品目标用户

### 2.1 核心目标用户
- 成年大众用户
- 对自然、地理、动物、视觉叙事感兴趣的人
- 不要求专业知识背景

### 2.2 次级用户
- 青少年用户
- 教育展示场景用户
- 作品集浏览者
- 科普传播场景用户

### 2.3 用户感受目标
用户不应该觉得自己在使用“工具”。  
用户应该觉得自己在：
- 看一张会呼吸的地图
- 翻阅一本自然绘本
- 观察一个大陆上的生命故事

---

## 3. 产品核心原则

### 3.1 顶层原则
1. **地图是页面本体**
2. **时间轴是主控制器**
3. **全国默认态是主体验**
4. **区域聚焦是镜头推进，不是切页**
5. **优先做科普叙事，不做分析型展示**
6. **优先做亲和与故事感，不做科技大屏**
7. **MVP 必须建立强边界，防止项目失控**

### 3.2 设计原则
1. 直觉优先：用户应自然知道“点击地图区域”
2. 视觉优先：一眼看过去像绘本，不像 dashboard
3. 情绪优先：要有治愈感、生命感和故事感
4. 可信优先：推断 / 事实 / 预测要分层
5. 性能优先：所有动画都要服务体验，不能为了炫技牺牲流畅度

## 4. 视觉方向与网页风格

### 4.1 总体风格
整体风格定为：

**治愈系互动自然绘本 + 手绘地图 + 纸质界面 + 温柔叙事**

### 4.2 必须体现的视觉关键词
- 羊皮纸
- 水彩感
- 手绘线条
- 低饱和大地色
- 可爱但不过幼
- 柔和圆角
- 温暖阴影
- 轻插画感
- 生命感
- 亲和
- 有故事的地图

### 4.3 明确禁止的视觉方向
禁止以下方向：
- 深海蓝 / 纯黑科技风
- 赛博朋克
- BI 数据大屏
- 典型 SaaS 后台风
- 强分析型 GIS 视觉
- 过度写实动物纪录片风
- 高饱和儿童玩具风

### 4.4 色彩规范
#### 基础色
- 羊皮纸米色
- 温暖沙色
- 浅草绿
- 柔和土黄
- 浅棕
- 暖灰白

#### 高亮色
- 柔和橘色
- 薄荷绿
- 天空蓝
- 雨林青
- 海龟绿

#### 色彩要求
- 饱和度克制
- 对比适中
- 不刺眼
- 不冷硬

### 4.5 地图底图风格
- 采用 SVG 手绘风地图
- 看起来像一张放在桌面上的绘本内页
- 地图边缘可以有轻微纸张纹理或旧纸质感
- 海域不是深色大海，而是浅色纸面上的柔和海域表达
- 州边界存在，但不应僵硬强烈

### 4.6 UI 面板风格
- 大圆角
- 暖白 / 米色浮层
- 柔和阴影
- 半拟物纸片感
- 轻微透明感可选，但不走玻璃拟态科技风

## 6. 页面信息架构

### 6.1 页面核心结构
这是一个单页网页，页面主要由以下区域构成：

#### A. 地图主舞台（页面主体）
- 占据页面绝大部分空间
- 地图就是页面本体
- 全国态与区域态都在这一层完成

#### B. 底部时间轴
- 固定于底部
- 始终可见
- 是主控制器

#### C. 顶部轻量浮层
展示：
- Logo / 标题
- 当前年份
- 当前层级标签
- 搜索框
- 返回全国提示（可选）

#### D. 右侧或浮动详情层
根据当前状态展示：
- 区域简介
- 动物详情
- 当前年份叙事说明
- 音频 / 图片 / 进度条

### 6.2 主体验规则
页面中不允许出现大量旁侧模块把地图压缩成“中间小块”。  
地图必须始终是视觉第一主角。

## 7. 时间轴设计

### 7.1 时间轴定位
时间轴是全站主控制器，不是辅助组件。

### 7.2 时间范围
MVP 采用以下时间逻辑：

- `1788前`：自然状态（推断）
- `1788`
- `1900`
- `1935` / `1936` 等个别重要事件点（如甘蔗蟾蜍引入、袋狼灭绝）
- `1950`
- `Now`

说明：
- 用户层支持逐年拖动
- 但底层依然是关键帧 + 插值
- 不要求每年都有真实独立数据表

### 7.3 时间层标签
- 推断层
- 事实层
- Now（探索层）

预测层本版可保留结构，但 MVP 不是强制主内容。

### 7.4 时间轴交互要求
- 可拖动
- 可播放 / 暂停
- 可回到起点
- 可跳到 Now
- 当前年份必须始终清晰可见
- 节点切换要有自然缓动
- 拖动时地图应实时变化，不允许明显延迟

### 7.5 时间轴视觉风格
- 像绘本页脚的时间丝带
- 而不是浏览器默认 slider
- 应有温暖质感
- 关键节点可用小图钉 / 小纸签式标识

## 8. 序章动画设计

### 8.1 是否必须
必须。

### 8.2 时长
不超过 10 秒。

### 8.3 目标
1. 建立绘本氛围
2. 教用户理解地图会随时间变化
3. 快速进入主体验，不拖沓

### 8.4 动画概念
“翻开一本自然绘本”

### 8.5 建议脚本
1. 页面像纸张缓慢展开
2. 澳大利亚地图像插画页出现在中央
3. 时间轴轻轻滑动，地图上的动物开始变化
4. 文案简短提示：
   - “拖动时间，看见这片大陆上的生命故事”
5. 动画结束，进入可操作状态

### 8.6 交互要求
- 首次自动播放
- 支持跳过
- 后续默认不重复播放
- 可在菜单中重新查看

## 9. 地图交互设计

### 9.1 默认全国态
用户进入网页后默认看到全国地图。  
全国态应直接展示：
- 多种动物在不同生态区域的存在感
- 迁徙 / 扩张 / 灭绝 / 保留的整体变化

### 9.2 区域点击
- 用户直接点击地图区域即可放大
- 不依赖文字按钮切换州
- Hover 时区域有轻微高亮提示
- 点击海域空白返回全国

### 9.3 放大体验
点击州区域后：
- 镜头平滑推进
- 带轻微弹性回弹感
- 区域内动物分布更清晰
- 其他州弱化
- 不跳转新页面

### 9.4 搜索
搜索只承担辅助聚焦作用：
- 输入动物名称
- 高亮该动物
- 弱化其他动物
- 不切换页面

### 9.5 选中动物
选中动物后：
- 地图中该动物成为前景对象
- 展示故事与说明
- 非 Now 时更强调时间变化
- Now 时切换为完整详情卡

#### D. 外来入侵与扩张史

- 区域归属

### 10.4 内容语气
趣味故事文本应：
- 口语化
- 有画面感
- 适合大众阅读
- 不要写成百科词条
- 不要写成学术报告

## 11. 动物可视化渲染机制（重点）

### 11.1 明确结论
最终主表达不使用：
- emoji
- 热力图
- 生硬椭圆框
- 分析型密度图

### 11.2 推荐主表达
采用：

**2D 卡通 / 手绘插画动物 + 生态拼贴式分布 + 迁徙丝带 + 事件状态变化**

### 11.3 地图中的动物表现层
#### A. 动物插画 Marker
- 统一风格的 2D 卡通 / 手绘动物
- 禁止原生 emoji
- Marker 应该有轻微呼吸感 / 悬浮感

#### B. 生态存在感层
用于表达某物种在区域中的“存在感”，不是分析边界。
可采用：
- 叶片拼贴
- 小脚印
- 草丛贴片
- 水波贴片
- 沙点贴片
- 海洋弧带

目的：
- 让地图更像绘本
- 让分布表达更亲和
- 不像工具型图表

#### C. 迁徙丝带层
只给迁徙型 / 扩张型物种使用。
例如：
- 南露脊鲸
- 绿海龟
- 甘蔗蟾蜍
- 欧洲野兔

表现要求：
- 卡通化
- 柔和
- 有方向感
- 不要硬箭头

#### D. 事件层
用于表达：
- 灭绝
- 区域消失
- 保护恢复
- 入侵扩张

### 11.4 状态表达规则
#### 繁盛态
- 色彩鲜明
- 插画清晰
- 有轻待机动效

#### 迁徙 / 扩张态
- 插画可做轻微平移
- 可以沿隐藏路径移动
- 可以出现新节点

#### 衰退态
- 色彩下降
- 透明度降低
- 分布贴片减少

#### 灭绝态
- 变为线稿
- 或半透明幽灵态
- 像铅笔擦除一样淡出
- 袋狼必须是明显的时间情绪节点

## 12. Now 时点信息卡

### 12.1 展示时机
仅当时间轴位于 Now，且用户点击了某物种时。

### 12.2 必须展示的内容
- 中文名
- 英文名
- 学名
- 绘本插图或实景图
- 简介
- 趣味故事
- 音频播放器
- 音频标签（真实录音 / AI 模拟）
- 濒危状态进度条
- 当前状态说明

### 12.3 濒危状态表现
采用进度条或等级条表示：
- 无危
- 近危
- 易危
- 濒危
- 极危
- 灭绝

要求：
- 色彩柔和

## 15. 技术实现建议（给 Codex）

### 15.1 推荐技术栈
- Next.js
- React
- TypeScript
- Tailwind CSS
- SVG
- Framer Motion

### 15.2 本期为什么以 SVG 为主
原因：
1. 手绘地图和绘本风格更容易精细控制
2. 动物插画与纸质 UI 更适合 SVG / DOM 混合表达
3. MVP 不要求真实 GIS 精度
4. 更适合做局部物理动画和手绘路径动画

### 15.3 动画库要求
使用 Framer Motion 或等价能力实现：
- 轻弹性出现
- 平滑镜头推进
- 面板滑出
- 时间轴联动的视觉过渡

### 15.4 性能边界（重要）
18 种动物会带来较大渲染压力，Codex 必须采取以下措施：
1. **同屏实例数控制**
   - 不允许无限制堆 Marker
   - 同类动物应采用“聚合表现 + 少量代表实例”

2. **节流与去抖**
   - 时间轴拖动时对昂贵渲染做 throttle

3. **关键帧插值**
   - 仅对关键年份做显著变化
   - 中间年份平滑过渡

4. **动画层分级**
   - 默认态轻动效
   - 聚焦态增强动效
   - 不要全图同时剧烈运动

5. **灭绝特效简化**
   - “线稿化 / 褪色 / 擦除”应简洁，不要制作高成本粒子系统

## 19. 验收标准

### 19.1 视觉验收
1. 页面第一眼必须像绘本，不像 dashboard
2. 地图必须是主舞台
3. 色彩必须温暖、柔和、治愈
4. 绝对不能残留深色科技感

### 19.2 交互验收
1. 用户应自然理解“点地图区域放大”
2. 时间轴拖动要顺滑
3. 面板和镜头切换带轻弹性
4. 不允许明显生硬切换

### 19.3 叙事验收
1. 袋狼灭绝要有清晰情绪节点
2. 甘蔗蟾蜍扩张要能被看懂
3. 树袋熊收缩要能被看懂
4. Now 卡片应有足够阅读吸引力

### 19.4 性能验收
1. 同屏展示 18 种核心物种时不应明显掉帧
2. 时间轴拖动不卡顿
3. 放大缩小不卡顿
4. 详情面板展开不卡顿

## 20. 对 Codex 的明确执行指令

1. 优先把项目做成“地图即页面”的绘本式体验。
2. 不要把它做成分析型产品。
3. 不要引入多余功能。
4. 不要为了将来扩展，过度设计当前架构。
5. 视觉气质统一比功能堆砌更重要。
6. 动画必须克制而有生命感。
7. 18 种动物是本期上限，不再继续扩容。
8. 默认态优先做全国演化，区域态再做聚焦阅读。
9. 本期数据全部本地化，不依赖真实 API。
10. 最终交付必须像一个可以上线展示的科普网页，而不是实验性 demo。

---

## 21. 最终产品结论

本 PRD 融合后的产品，不再是“自然博物馆式深色地图”，而是：

**一张会随着时间呼吸的澳大利亚自然绘本地图。**

它的核心不是数据分析，而是把下面这件事讲清楚：

- 曾经这里生活着谁
- 它们怎样移动
- 它们怎样消失
- 今天还能在哪里看见它们

这是一张地图，也是一页绘本，更是一种带有温度的科普叙事体验。
